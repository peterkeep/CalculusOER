    Subsection 4.6.1 Linearly Approximating a Function

    The visual above should provide us with a nice framework to think about how we might approximate a function linearly, but we can recap some basic ideas:

    • Our functions only look "locally linear" when we zoom in around some single point. Another way of saying this is that our tangent line only matches the behavior of our function really close to the point where we built the tangent line.
    • We have a kind of vague or ambiguous idea of accuracy in approximation.     Definition 4.6.1. Linear Approximation of a Function.

    If \(f(x)\) is differentiable at \(x=a\text{,}\) then we say that a linear approximation of \(f(x)\) centered at \(x=a\) is:
    \begin{equation*} L(x) = f'(a)(x-a)+f(a)\text{.} \end{equation*}
    We know, then, that \(L(x)\approx f(x)\) for \(x\)-values "close" to the center, \(x=a\text{.}\)
    Note that the center is just the point at which we are building this linear approximation: the point at which we build the tangent line.

    Activity 4.6.3. Walking in the Footsteps of Ancient Mathematicians.

    Letâ€™s travel all the way back to the first (or maybe second) century AD and recreate Heronâ€™s method to approximate the value of \(\sqrt{2}\text{.}\) Weâ€™ll develop this using modern calculus, and simple linear approximation.
    Weâ€™re going to re-frame the problem, and instead weâ€™re going to try to use a linear approximation of \(f(x)=x^2-2\) to approximate the \(x\)-value where \(f(x)=0\text{.}\) We know enough about quadratic functions to know that there are two values: \(x=-\sqrt{2}\) and \(x=\sqrt{2}\text{.}\)
    (a)
    Weâ€™re going to build a linear approximation of \(f(x) = x^2-2\text{,}\) and we need a reasonable center.     Find \(f'(x)\text{,}\) and then construct the linear approximation:
    \begin{equation*} L(x) = f'(2)(x-2)+f(2)\text{.} \end{equation*}
    Solution.
    Since \(f'(x)=2x\text{,}\) we have \(f'(2)=4\) and \(f(2) = 2\text{.}\) So then we end up with the following for our linear approximation:
    \begin{equation*} L(x) = 4(x-2)+2\text{.} \end{equation*}
    (b)
    Now we know that \(L(x)\approx f(x)\) for \(x\)-values near our center, \(x=2\text{.}\) What if we estimate the \(x\)-value where \(f(x)=0\) by solving \(L(x)=0\) instead? Since \(L(x)\approx f(x)\text{,}\) the \(x\)-value where \(L(x)=0\) should make \(f(x)\) pretty close to 0 at least.
    Solve \(L(x)=0\text{.}\)
    Solution.
    \begin{align*} 0 \amp = 4(x-2)+2\\ -2 \amp 4(x-2)\\ -\frac{2}{4} \amp x-2\\ 2 - \frac{1}{2} \amp x\\ x \amp = \frac{3}{2} \end{align*}
    (c)
    Ok, this might be kind of close to the value of \(\sqrt{2}\text{,}\) right? Letâ€™s visualize this.
    Hm...so this isnâ€™t that good of an approximation yet. We can check this by looking at the actual value of our function at \(x=\frac{3}{2}\) and seeing if itâ€™s close to 0.
    \begin{align*} f\left(\frac{3}{2}\right) \amp = \left(\frac{3}{2}\right)^2 - 2\\ \amp = \frac{9}{4} - 2\\ \amp \frac{1}{4} \end{align*}
    This...isnâ€™t that close to 0.

    Solution.
    \begin{align*} L(x) \amp f'\left(\frac{3}{2}\right)\left(x-\frac{3}{2}\right)+f\left(\frac{3}{2}\right) \\ \amp = 3\left(x-\frac{3}{2}) + \frac{1}{4} \end{align*}

    Solution.
    \begin{align*} 0 \amp = 3\left(x-\frac{3}{2}) + \frac{1}{4}\\ -\frac{1}{4} \amp = 3\left(x-\frac{3}{2}\right)\\ -\frac{1}{12} \amp = x- \frac{3}{2}\\ \frac{3}{2} - \frac{1}{12} \amp \\ x \amp= \frac{17}{12} \end{align*}
    (e)
    We can keep repeating this process, and thatâ€™s exactly what the mathematicians we talked about discovered.
    Say weâ€™ve build a linear approximation at some \(x\)-value (weâ€™ll call it \(x_{\text{old}}\)).

    (f)
    Letâ€™s visualize these calculations.
    Something kind of strange happens in the last two steps. Why does the value of our estimation not change? What happens to our estimate?

    Definition 4.6.2. Newtonâ€™s Method for Approximating Zeros of Functions.

    If \(x_0\) is some initial estimation of a solution to \(f(x)=0\text{,}\) then we we can iteratively generate more estimations using the following formula:
    \begin{equation*} x_n = x_{n-1} - \frac{f(x_{n-1})}{f'(x_{n-1})} \end{equation*}
    provided that \(f'(x_{n-1})\) exists and is non-zero.
    A good question to ask is about when this process stops. If we want to estimate some \(x\)-intercept of a function, like \(\sqrt{2}\) in ActivityÂ 4.6.3, then how many steps is enough? There are a couple of ways we can approach this:
    1. We can just state at the beginning how many iterations weâ€™re going to do. This is what happened in ActivityÂ 4.6.3, since this activity was written to only make you calculate a specific number of these estimations. We could have started by saying that weâ€™ll calculate this 3 times, or maybe 100 times.
    2. We can test to see what \(f(x_n)\) is, and then stop when it is within some pre-determined distance from 0. We also did this when we noticed that \(f\left(\frac{3}{2}\right)=\frac{1}{4}\) was not very close to 0 (after our first estimation), and so we should calculate this again. We could start by saying that weâ€™ll continue until we see a \(y\)-value that is within 0.0001 of 0, or some other small distance.
    3. We can test to see how close our approximations are to each other, and stop when theyâ€™re close enough. We saw this happen in the visualization: the last two estimates were the same! They actually werenâ€™t, but since the applet only displayed 4 decimal places, the numbers appeared the same after rounding. Maybe we set some criteria there, or we look at the distance between \(x_n\) and \(x_{n-1}\) (two successive estimates) and stop when they are within some distance from each other.
    In reality, we often choose a combination of these. Maybe we set distance threshold for stopping, but use a maximum of 10 iterations as a backup plan. This happens often when we code this algorithm and have a computer run it. It is possible for this code to never give us two successive estimates that are close enough to stop, and so the code would run forever unless we cut it off at 100 iterations or some other value.
    A wonderful thing about this small process is that, while it is ancient (dating back to Heron in the first or second century), it is still used today. This is a powerful estimation method that can be used in a variety of areas including statistics and data science.
